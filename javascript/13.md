## 13. 스코프

스코프(scope)는 식별자가 유효한 범위를 말한다. 변수, 함수, 클래스 같은 모든 식별자는 자신이 선언된 위치에 따라 참조 가능한 범위가 결정된다.

## 13.1 스코프란?

함수의 매개변수는 함수 몸체 내부에서만 참조할 수 있다. 이것은 매개변수의 스코프가 함수 몸체 내부로 한정되기 때문이다.

```js
function add(x, y) {
  console.log(x, y); // 2 5
  return x + y;
}

add(2, 5);

// console.log(x, y); // ReferenceError
```

변수도 선언된 위치에 따라 참조 가능한 범위가 달라진다.

```js
var x = "global";

function foo() {
  var x = "local";
  console.log(x);
}

foo();          // local
console.log(x); // global
```

위 예제에는 이름이 같은 `x`가 두 개 있다. 자바스크립트 엔진은 스코프를 기준으로 어떤 `x`를 참조할지 결정한다. 이를 **식별자 결정**이라고 한다.

즉, 스코프는 자바스크립트 엔진이 식별자를 검색할 때 사용하는 규칙이다.

### 스코프는 네임스페이스다

스코프가 없다면 같은 이름의 변수는 프로그램 전체에서 하나만 사용할 수 있다. 하지만 스코프가 있기 때문에 서로 다른 범위에서는 같은 이름의 식별자를 사용할 수 있다.

```js
var x = "global";

function foo() {
  var x = "local";
  console.log(x);
}
```

스코프 내에서는 식별자가 유일해야 하지만, 다른 스코프에서는 같은 이름의 식별자를 사용할 수 있다.

### var와 중복 선언

`var` 키워드로 선언한 변수는 같은 스코프 안에서 중복 선언이 허용된다.

```js
function foo() {
  var x = 1;
  var x = 2;

  console.log(x); // 2
}

foo();
```

반면 `let`, `const`는 같은 스코프 안에서 중복 선언을 허용하지 않는다.

```js
function bar() {
  let x = 1;
  // let x = 2; // SyntaxError
}
```

## 13.2 스코프의 종류

코드는 크게 전역과 지역으로 구분할 수 있다.

| 구분 | 설명 | 스코프 | 변수 |
|------|------|--------|------|
| 전역 | 코드의 가장 바깥 영역 | 전역 스코프 | 전역 변수 |
| 지역 | 함수 몸체 내부 | 지역 스코프 | 지역 변수 |

### 13.2.1 전역과 전역 스코프

전역에 선언된 변수는 전역 스코프를 갖는다. 전역 변수는 어디서든 참조할 수 있다.

```js
var x = "global x";
var y = "global y";

function outer() {
  console.log(x); // global x
  console.log(y); // global y
}

outer();
```

### 13.2.2 지역과 지역 스코프

지역은 함수 몸체 내부를 말한다. 지역에 선언된 변수는 지역 스코프를 갖고, 자신이 선언된 지역과 하위 지역에서만 참조할 수 있다.

```js
var x = "global x";

function outer() {
  var z = "outer local z";

  function inner() {
    var x = "inner local x";

    console.log(x); // inner local x
    console.log(z); // outer local z
  }

  inner();
}

outer();
// console.log(z); // ReferenceError
```

지역 변수는 자신이 선언된 함수 내부와 그 하위 함수에서 유효하다.

## 13.3 스코프 체인

함수는 중첩될 수 있고, 함수가 중첩되면 스코프도 계층적으로 중첩된다. 이렇게 스코프가 계층적으로 연결된 것을 **스코프 체인**이라고 한다.

```js
var x = "global";

function outer() {
  var y = "outer";

  function inner() {
    var z = "inner";

    console.log(x); // global
    console.log(y); // outer
    console.log(z); // inner
  }

  inner();
}

outer();
```

자바스크립트 엔진은 변수를 참조할 때 현재 스코프에서 시작해 상위 스코프 방향으로 이동하며 식별자를 검색한다.

검색 순서:

1. 현재 스코프에서 식별자를 찾는다.
2. 없으면 상위 스코프로 이동한다.
3. 전역 스코프까지 검색한다.
4. 끝까지 없으면 `ReferenceError`가 발생한다.

하위 스코프로 내려가며 검색하지는 않는다. 따라서 상위 스코프의 변수는 하위 스코프에서 참조할 수 있지만, 하위 스코프의 변수는 상위 스코프에서 참조할 수 없다.

### 13.3.1 스코프 체인에 의한 변수 검색

```js
var x = "global x";
var y = "global y";

function outer() {
  var z = "outer local z";

  function inner() {
    var x = "inner local x";

    console.log(x); // inner local x
    console.log(y); // global y
    console.log(z); // outer local z
  }

  inner();
}

outer();
```

- `x`: `inner` 스코프에서 먼저 찾고 발견하므로 `inner local x`
- `y`: `inner` → `outer` → 전역 순서로 찾고 전역에서 발견
- `z`: `inner` → `outer` 순서로 찾고 `outer`에서 발견

### 13.3.2 스코프 체인에 의한 함수 검색

함수도 식별자에 함수 객체가 할당된 것이므로 스코프를 갖는다. 함수 호출 시에도 스코프 체인을 따라 함수 식별자를 검색한다.

```js
function foo() {
  console.log("global function foo");
}

function bar() {
  function foo() {
    console.log("local function foo");
  }

  foo();
}

bar(); // local function foo
```

`bar` 내부의 `foo()` 호출은 먼저 `bar`의 지역 스코프에서 `foo`를 찾기 때문에 지역 함수가 호출된다.

## 13.4 함수 레벨 스코프

대부분의 프로그래밍 언어는 모든 코드 블록(`if`, `for`, `while`, `try/catch` 등)이 지역 스코프를 만든다. 이를 **블록 레벨 스코프**라고 한다.

하지만 `var` 키워드로 선언한 변수는 함수 몸체만 지역 스코프로 인정한다. 이를 **함수 레벨 스코프**라고 한다.

```js
var x = 1;

if (true) {
  var x = 10;
}

console.log(x); // 10
```

`if` 블록 안에서 선언한 `var x`는 블록 지역 변수가 아니라 전역 변수다. 그래서 기존 전역 변수 `x`가 의도치 않게 변경된다.

`for` 문에서도 같은 문제가 생길 수 있다.

```js
var i = 10;

for (var i = 0; i < 5; i++) {
  console.log(i); // 0 1 2 3 4
}

console.log(i); // 5
```

`let`, `const`는 블록 레벨 스코프를 지원하므로 이런 문제를 줄일 수 있다.

## 13.5 렉시컬 스코프

함수의 상위 스코프를 결정하는 방식에는 두 가지가 있다.

1. 함수를 어디서 호출했는지에 따라 상위 스코프를 결정하는 방식: 동적 스코프
2. 함수를 어디서 정의했는지에 따라 상위 스코프를 결정하는 방식: 렉시컬 스코프

자바스크립트는 렉시컬 스코프를 따른다.

```js
var x = 1;

function foo() {
  var x = 10;
  bar();
}

function bar() {
  console.log(x);
}

foo(); // 1
bar(); // 1
```

`bar` 함수는 전역에서 정의되었다. 따라서 `bar`의 상위 스코프는 호출 위치와 관계없이 전역 스코프다.

즉, 자바스크립트 함수의 상위 스코프는 **호출 위치가 아니라 정의 위치**에 의해 결정된다.

렉시컬 스코프는 이후 클로저를 이해하는 핵심 기반이다.

---

## 내용 정리

- 스코프는 식별자가 유효한 범위다.
- 자바스크립트 엔진은 스코프를 기준으로 어떤 식별자를 참조할지 결정한다.
- 전역 스코프에서 선언된 전역 변수는 어디서든 참조할 수 있다.
- 지역 스코프에서 선언된 지역 변수는 자신이 선언된 지역과 하위 지역에서만 참조할 수 있다.
- 스코프 체인은 현재 스코프에서 상위 스코프로 이어지는 식별자 검색 경로다.
- `var`는 함수 레벨 스코프를 따르므로 블록 내부 선언이 전역/함수 스코프에 영향을 줄 수 있다.
- `let`, `const`는 블록 레벨 스코프를 지원한다.
- 자바스크립트는 렉시컬 스코프를 따르며, 함수의 상위 스코프는 정의 위치로 결정된다.

## 의문점 제기

1. 전역 변수가 많아질수록 식별자 충돌과 추적 문제가 커지는데, 이를 구조적으로 줄이는 방법은 무엇일까?
2. `var`의 함수 레벨 스코프는 어떤 과거 호환성 때문에 유지되고 있을까?
3. `let`, `const`를 기본으로 사용하면 스코프 관련 버그를 얼마나 줄일 수 있을까?
4. 스코프 체인 검색이 길어지면 성능에 의미 있는 영향을 줄 수 있을까?
5. 중첩 함수가 상위 스코프의 변수를 참조하는 구조는 언제 클로저로 이어질까?
6. 렉시컬 스코프와 동적 스코프의 차이를 실제 디버깅 상황에서 어떻게 체감할 수 있을까?
7. 같은 이름의 변수를 내부 스코프에서 다시 선언하는 섀도잉은 언제 허용하고 언제 피해야 할까?
8. 함수 선언문이 스코프 체인에서 검색되는 방식은 변수 검색과 어떤 점에서 같고 어떤 점에서 다를까?
