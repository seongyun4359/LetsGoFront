## 9. 타입 변환과 단축 평가

## 9.1 타입 변환이란?

자바스크립트의 모든 값은 타입을 가진다. 값의 타입은 개발자의 의도에 따라 다른 타입으로 변환될 수 있다.

- **명시적 타입 변환**: 개발자가 의도적으로 타입을 변환하는 것
- **암묵적 타입 변환**: 자바스크립트 엔진이 표현식을 평가하는 과정에서 자동으로 타입을 변환하는 것

```js
var x = 10;

var str = x.toString(); // 명시적 타입 변환
console.log(typeof str, str); // string 10
console.log(typeof x, x);     // number 10

var str2 = x + ""; // 암묵적 타입 변환
console.log(typeof str2, str2); // string 10
```

타입 변환은 기존 원시 값을 직접 바꾸는 것이 아니다. 기존 값을 바탕으로 **새로운 타입의 값**을 만들어 사용하는 것이다.

중요한 것은 명시적/암묵적 변환 중 무엇이 항상 옳은지가 아니라, 코드의 결과를 예측할 수 있어야 한다는 점이다.

## 9.2 암묵적 타입 변환

자바스크립트 엔진은 표현식을 평가할 때 문맥에 맞게 값을 자동 변환한다.

```js
"10" + 2; // "102" - 문자열 문맥
5 * "10"; // 50    - 숫자 문맥
!0;       // true  - 불리언 문맥
```

### 9.2.1 문자열 타입으로 변환

`+` 연산자는 피연산자 중 하나 이상이 문자열이면 문자열 연결 연산자로 동작한다. 이때 문자열이 아닌 값은 문자열로 암묵적 변환된다.

```js
1 + "2"; // "12"
```

템플릿 리터럴의 표현식 삽입도 평가 결과를 문자열로 변환한다.

```js
`1 + 1 = ${1 + 1}`; // "1 + 1 = 2"
```

대표적인 문자열 변환 결과:

```js
0 + "";          // "0"
-0 + "";         // "0"
NaN + "";        // "NaN"
Infinity + "";   // "Infinity"
true + "";       // "true"
false + "";      // "false"
null + "";       // "null"
undefined + "";  // "undefined"

({}) + "";       // "[object Object]"
[] + "";         // ""
[10, 20] + "";   // "10,20"
```

단, `Symbol()`은 문자열로 암묵적 변환할 수 없어 `TypeError`가 발생한다.

### 9.2.2 숫자 타입으로 변환

산술 연산자는 숫자 값을 만들어야 하므로 피연산자를 숫자로 변환한다.

```js
1 - "1";   // 0
1 * "10";  // 10
1 / "one"; // NaN
```

비교 연산자도 크기를 비교해야 하므로 숫자 타입으로 변환하는 경우가 있다.

```js
"1" > 0; // true
```

단항 `+` 연산자는 값을 숫자 타입으로 변환한다.

```js
+"";          // 0
+"0";         // 0
+"1";         // 1
+"string";    // NaN
+true;        // 1
+false;       // 0
+null;        // 0
+undefined;   // NaN
+[];          // 0
+{};          // NaN
```

주의할 변환:

- `""`, `[]`, `null`, `false` → `0`
- `true` → `1`
- `undefined`, 일반 객체, 숫자로 바꿀 수 없는 문자열 → `NaN`

### 9.2.3 불리언 타입으로 변환

조건문이나 삼항 조건 연산자의 조건식은 불리언으로 평가되어야 한다. 자바스크립트는 이때 값을 `Truthy` 또는 `Falsy`로 구분한다.

Falsy 값:

- `false`
- `undefined`
- `null`
- `0`, `-0`
- `NaN`
- `""` 빈 문자열

Falsy 값을 제외한 모든 값은 Truthy 값이다.

```js
if ("") console.log("1");
if (true) console.log("2");
if (0) console.log("3");
if ("str") console.log("4");
if (null) console.log("5");

// 2
// 4
```

Truthy/Falsy 판별:

```js
function isFalsy(value) {
  return !value;
}

function isTruthy(value) {
  return !!value;
}

isFalsy(false);     // true
isFalsy(undefined); // true
isFalsy(null);      // true
isFalsy(0);         // true
isFalsy(NaN);       // true
isFalsy("");        // true

isTruthy("0");      // true
isTruthy({});       // true
isTruthy([]);       // true
```

## 9.3 명시적 타입 변환

명시적 타입 변환은 개발자가 의도를 드러내고 타입을 바꾸는 방법이다.

주요 방법:

- 표준 빌트인 생성자 함수(`String`, `Number`, `Boolean`)를 `new` 없이 호출
- 빌트인 메서드 사용
- 암묵적 타입 변환 패턴을 의도적으로 사용

### 9.3.1 문자열 타입으로 변환

```js
String(1);        // "1"
String(NaN);      // "NaN"
String(Infinity); // "Infinity"
String(true);     // "true"
String(false);    // "false"

(1).toString();        // "1"
(NaN).toString();      // "NaN"
(Infinity).toString(); // "Infinity"
(true).toString();     // "true"

1 + "";        // "1"
NaN + "";      // "NaN"
true + "";     // "true"
```

### 9.3.2 숫자 타입으로 변환

```js
Number("0");     // 0
Number("-1");    // -1
Number("10.53"); // 10.53
Number(true);    // 1
Number(false);   // 0

parseInt("0");        // 0
parseInt("-1");       // -1
parseFloat("10.53");  // 10.53

+"0";      // 0
+"-1";     // -1
+"10.53";  // 10.53
+true;     // 1
+false;    // 0

"10" * 1;  // 10
true * 1;  // 1
false * 1; // 0
```

### 9.3.3 불리언 타입으로 변환

```js
Boolean("x");     // true
Boolean("");      // false
Boolean("false"); // true
Boolean(0);       // false
Boolean(1);       // true
Boolean(NaN);     // false
Boolean(null);    // false
Boolean(undefined); // false
Boolean({});      // true
Boolean([]);      // true

!!"x";       // true
!!"";        // false
!!"false";   // true
!!0;         // false
!!1;         // true
!!NaN;       // false
!!null;      // false
!!undefined; // false
!!{};        // true
!![];        // true
```

## 9.4 단축 평가

### 9.4.1 논리 연산자를 사용한 단축 평가

`&&`, `||` 연산자는 항상 불리언 값을 반환하지 않는다. 논리 연산 결과를 결정하는 피연산자를 그대로 반환한다.

```js
"Cat" && "Dog"; // "Dog"
"Cat" || "Dog"; // "Cat"
```

단축 평가는 표현식을 평가하는 도중 결과가 확정되면 나머지 평가를 생략하는 것이다.

| 표현식 | 평가 결과 |
|--------|-----------|
| `true || anything` | `true` |
| `false || anything` | `anything` |
| `true && anything` | `anything` |
| `false && anything` | `false` |

실제 값 기준:

```js
"Cat" || "Dog";   // "Cat"
false || "Dog";   // "Dog"
"Cat" || false;   // "Cat"

"Cat" && "Dog";   // "Dog"
false && "Dog";   // false
"Cat" && false;   // false
```

#### if 문 대체

조건이 Truthy일 때만 실행/할당하고 싶다면 `&&`를 사용할 수 있다.

```js
var done = true;
var message = "";

message = done && "완료";
console.log(message); // 완료
```

조건이 Falsy일 때 기본값을 사용하고 싶다면 `||`를 사용할 수 있다.

```js
var done = false;
var message = "";

message = done || "미완료";
console.log(message); // 미완료
```

단, 단순한 조건 분기에는 좋지만 복잡한 로직까지 억지로 넣으면 가독성이 떨어질 수 있다.

#### 객체 프로퍼티 접근 전 방어

객체가 `null` 또는 `undefined`일 수 있을 때 단축 평가로 에러를 피할 수 있다.

```js
var elem = null;

// var value = elem.value; // TypeError
var value = elem && elem.value;

console.log(value); // null
```

#### 함수 매개변수 기본값

```js
function getStringLength(str) {
  str = str || "";
  return str.length;
}

getStringLength();     // 0
getStringLength("hi"); // 2
```

ES6 이후에는 기본 매개변수를 사용하는 편이 더 명확하다.

```js
function getStringLength(str = "") {
  return str.length;
}
```

### 9.4.2 옵셔널 체이닝 연산자

옵셔널 체이닝 연산자 `?.`는 좌항이 `null` 또는 `undefined`이면 `undefined`를 반환하고, 그렇지 않으면 프로퍼티 참조를 이어간다.

```js
var elem = null;

var value = elem?.value;
console.log(value); // undefined
```

기존 `&&` 단축 평가와 차이:

```js
var str = "";

var length1 = str && str.length;
console.log(length1); // ""

var length2 = str?.length;
console.log(length2); // 0
```

`&&`는 좌항이 Falsy이면 그대로 반환하지만, `?.`는 `null` 또는 `undefined`만 막고 `0`, `""`, `false` 같은 값은 정상 값으로 취급한다.

### 9.4.3 null 병합 연산자

null 병합 연산자 `??`는 좌항이 `null` 또는 `undefined`이면 우항을 반환하고, 그렇지 않으면 좌항을 반환한다.

```js
var foo = null ?? "default string";
console.log(foo); // default string
```

기존 `||` 기본값 처리와 차이:

```js
var value1 = "" || "default string";
console.log(value1); // default string

var value2 = "" ?? "default string";
console.log(value2); // ""
```

`||`는 Falsy 값 전체를 기본값으로 대체하지만, `??`는 `null`과 `undefined`만 대체한다.  
따라서 `0`, `""`, `false`가 유효한 값이라면 `??`를 사용하는 것이 안전하다.

---

## 내용 정리

- 타입 변환은 기존 값을 직접 바꾸는 것이 아니라 새로운 타입의 값을 만들어 사용하는 것이다.
- 명시적 타입 변환은 개발자의 의도가 드러나고, 암묵적 타입 변환은 문맥에 따라 엔진이 자동 수행한다.
- 문자열 문맥에서는 문자열 변환, 산술/비교 문맥에서는 숫자 변환, 조건식 문맥에서는 불리언 변환이 일어난다.
- Falsy 값은 `false`, `undefined`, `null`, `0`, `-0`, `NaN`, `""`이고, 그 외는 Truthy다.
- `&&`와 `||`는 불리언이 아니라 결과를 결정한 피연산자를 반환할 수 있다.
- `?.`는 `null`/`undefined`만 안전하게 막고 프로퍼티 접근을 이어간다.
- `??`는 `null`/`undefined`일 때만 기본값을 사용하므로 `0`, `""`, `false`를 보존한다.

## 의문점 제기

1. 명시적 타입 변환과 암묵적 타입 변환 중 어떤 것을 팀 컨벤션으로 우선해야 할까?
2. `+value`, `value * 1`, `Number(value)` 중 숫자 변환 방식은 어떤 기준으로 선택하는 것이 좋을까?
3. `||` 기본값 처리로 인해 `0`이나 빈 문자열이 사라지는 버그를 어떻게 예방할 수 있을까?
4. `?.`를 과도하게 사용하면 실제로는 반드시 있어야 할 데이터 누락을 숨길 위험은 없을까?
5. `??`와 함수 기본 매개변수는 어떤 상황에서 각각 더 적합할까?
6. Truthy/Falsy를 조건문에 그대로 쓰는 것과 명시 비교(`value !== null`)를 쓰는 기준은 무엇일까?
7. 단축 평가를 if 문 대체로 사용할 때 가독성을 해치지 않는 한계는 어디까지일까?
8. 타입 변환 규칙을 모르는 사람이 읽어도 안전한 코드를 만들려면 어떤 ESLint 규칙이 도움이 될까?
