## 18. 함수와 일급 객체

함수는 자바스크립트에서 값으로 취급된다.  
즉, 함수를 생성하고 저장하고 전달하고 반환할 수 있으며, 이 특성이 함수형 스타일의 기반이 된다.

## 18.1 일급 객체

일급 객체(First-class object)의 조건:

- 무명의 리터럴로 생성 가능
- 변수/자료구조(객체, 배열 등)에 저장 가능
- 함수의 인수로 전달 가능
- 함수의 반환값으로 사용 가능

함수는 위 조건을 모두 만족하므로 일급 객체다.

```js
const increase = function (num) {
  return ++num;
};

const decrease = function (num) {
  return --num;
};

const auxs = { increase, decrease };

function makeCounter(aux) {
  let num = 0;
  return function () {
    num = aux(num);
    return num;
  };
}

const increaser = makeCounter(auxs.increase);
const decreaser = makeCounter(auxs.decrease);
```

핵심:
- 함수는 값이므로 어디든 할당 가능
- 함수를 인수/반환값으로 다루면서 고차 함수 패턴 구현 가능

## 18.2 함수 객체의 프로퍼티

함수도 객체이므로 프로퍼티를 가진다. 다만 일반 객체와 달리 함수 고유 프로퍼티가 있다.

대표적으로 `arguments`, `caller`, `length`, `name`, `prototype`이 있으며,  
`__proto__`는 함수 고유 프로퍼티가 아니라 상속 프로퍼티다.

```js
function square(number) {
  return number * number;
}

console.log(Object.getOwnPropertyDescriptors(square));
```

### 18.2.1 arguments 프로퍼티

- 함수 호출 시 전달된 인수 정보를 담는 유사 배열 객체
- 함수 내부에서 지역 변수처럼 참조
- 매개변수 개수와 인수 개수가 달라도 인수 전체를 보관

```js
function sum() {
  let res = 0;
  for (let i = 0; i < arguments.length; i++) {
    res += arguments[i];
  }
  return res;
}
```

주의:
- `arguments`는 배열이 아니라 유사 배열 객체
- ES6 이후에는 `...rest` 파라미터 사용이 더 권장됨

```js
function sum(...args) {
  return args.reduce((pre, cur) => pre + cur, 0);
}
```

### 18.2.2 caller 프로퍼티

- 함수를 호출한 함수를 가리키는 비표준 프로퍼티
- 표준이 아니므로 실무 사용 비권장

### 18.2.3 length 프로퍼티

- 함수 정의 시 선언한 매개변수 개수
- `arguments.length`(전달된 인수 개수)와 다를 수 있음

```js
function foo() {}
function bar(x) {}
function baz(x, y) {}

console.log(foo.length); // 0
console.log(bar.length); // 1
console.log(baz.length); // 2
```

### 18.2.4 name 프로퍼티

- 함수 이름을 나타냄
- ES6에서 익명 함수 표현식의 `name`은 식별자명을 추론해 가질 수 있음

```js
var anonymousFunc = function () {};
console.log(anonymousFunc.name); // ES6+: "anonymousFunc"
```

### 18.2.5 __proto__ 접근자 프로퍼티

- 모든 객체가 상속받는 `Object.prototype`의 접근자 프로퍼티
- 함수 고유 프로퍼티가 아님
- 내부 슬롯 `[[Prototype]]`에 간접 접근하기 위한 통로

```js
const obj = { a: 1 };
console.log(obj.__proto__ === Object.prototype); // true
```

### 18.2.6 prototype 프로퍼티

- `constructor`인 함수 객체만 가지는 프로퍼티
- 생성자 호출(`new`) 시 만들어질 인스턴스의 프로토타입 객체를 가리킴

```js
(function () {}).hasOwnProperty("prototype"); // true
({}).hasOwnProperty("prototype"); // false
```

## 핵심 정리

- 함수는 일급 객체이므로 값처럼 자유롭게 다룰 수 있다.
- 함수 객체는 일반 객체와 달리 고유 프로퍼티(`arguments`, `length`, `name`, `prototype` 등)를 갖는다.
- `arguments`는 유사 배열 객체이며, 모던 문법에서는 `rest parameter`가 더 실용적이다.
- `length`는 매개변수 개수, `arguments.length`는 실제 인수 개수다.
- `__proto__`는 상속 프로퍼티이고, `prototype`은 생성자 함수의 핵심 프로퍼티다.

## 의문점 제기(생각해볼 질문)

- 함수가 일급 객체라는 사실이 콜백 지옥과 비동기 추상화에 각각 어떤 영향을 주었을까?
- `arguments` 대신 `...args`를 쓰면 디버깅/타입 추론/가독성에서 어떤 이점이 생길까?
- `function`과 화살표 함수는 `arguments`, `this`, `prototype`에서 어떤 본질적 차이가 있을까?
- 함수의 `length`를 이용한 API 설계(오버로딩 흉내)는 유지보수에 도움이 될까, 오히려 위험할까?
- `__proto__` 직접 사용보다 `Object.getPrototypeOf`를 권장하는 이유는 무엇일까?
- 생성자 함수의 `prototype` 체인을 잘못 설정하면 어떤 런타임 버그가 생길 수 있을까?

## 의문점 답변

- 함수가 일급 객체라서 콜백 패턴은 유연해졌지만, 제어 흐름 분산으로 콜백 지옥이 생겼다. 반대로 Promise/async 같은 비동기 추상화도 같은 특성 덕분에 가능해졌다.
- `...args`는 진짜 배열이라 메서드 사용이 쉽고 타입 추론/자동완성 품질이 좋아 디버깅과 가독성 모두 이점이 크다.
- 일반 `function`은 자체 `this`, `arguments`, `prototype`을 갖지만 화살표 함수는 렉시컬 `this`를 캡처하고 `arguments`/`prototype`이 없다.
- `length` 기반 오버로딩 흉내는 빠른 실험에는 유용하지만, 호출 규칙이 늘어나면 숨은 분기 버그가 커져 장기 유지보수에는 위험하다.
- `__proto__`는 레거시 접근자이고 일부 객체에서 없을 수 있어 이식성이 떨어진다. `Object.getPrototypeOf`는 명세적이고 예측 가능하다.
- `prototype` 체인을 잘못 연결하면 `instanceof` 오판정, `constructor` 깨짐, 상속 메서드 누락, 메서드 호출 시 `TypeError`가 발생할 수 있다.
