# 1. 프로그래밍

---

## 내용 정리

- 프로그래밍은 컴퓨터와의 커뮤니케이션이며, 시작점은 요구사항(문제) 명확화다.
- 핵심 역량은 단순 문법 암기가 아니라 문제 해결 능력이다.
- 사람의 모호한 표현은 컴퓨터가 이해하기 어려우므로 수치화, 명세화, 분해가 필요하다.
- 이를 위해 컴퓨팅 사고(Computational Thinking)가 필요하며, 문제를 작은 단위로 나누고 패턴화해야 한다.
- 사람은 기계어를 직접 다루기 어렵기 때문에 프로그래밍 언어와 컴파일러/인터프리터를 사용한다.
- 프로그래밍 언어는 구문(syntax)과 의미(semantics)로 이루어진다.
- 문법적으로 맞는 코드라도 의미적으로 틀릴 수 있다.
- 좋은 코드는 문법에 맞는 것을 넘어 요구사항을 정확히 실현해야 한다.
- 결국 프로그래밍은 요구사항을 자료구조, 함수, 제어흐름으로 변환하는 작업이다.

## 의문점 제기

1. 문제 해결 능력은 어떤 훈련(분해, 추상화, 모델링)으로 가장 효과적으로 키울 수 있을까?
2. 요구사항이 모호할 때 구현 전에 어떤 형식(입출력 표, 예외 케이스, 유저 스토리)으로 정리하면 좋을까?
3. 문법적으로 맞지만 의미적으로 틀린 코드를 줄이기 위해 JavaScript에서 어떤 도구(TypeScript, ESLint, 테스트)가 가장 효과적일까?
4. 컴파일러와 인터프리터의 차이는 JavaScript 실행 환경(V8, JIT)에서 어떻게 이해하면 정확할까?
5. 컴퓨터 관점의 사고를 실제 코드로 옮길 때, 어디까지 명시화해야 과하지 않을까?
6. 요구사항을 자료구조와 함수로 바꿀 때, 무엇을 먼저 설계해야 할까?
7. NaN 같은 런타임 의미 오류를 사전에 잡기 위한 최소 테스트 전략은 무엇일까?
8. 문제 해결 관점에서 성능, 가독성, 유지보수성의 우선순위는 상황에 따라 어떻게 달라질까?
