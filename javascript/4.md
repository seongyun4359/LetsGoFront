## 4. 변수

## 4.1 변수란 무엇인가? 왜 필요한가?

- 애플리케이션은 결국 **데이터를 입력받아 처리하고 결과를 출력**한다.
- 변수는 데이터를 저장하고 다시 읽어 쓰기 위한 핵심 메커니즘이다.
- 예를 들어 `10 + 20`을 계산하면 결과 `30`은 메모리 어딘가에 저장되지만, 메모리 주소를 직접 다루는 것은 위험하고 비현실적이다.
- 자바스크립트는 개발자가 메모리 주소를 직접 제어하지 않도록 하고, 대신 **변수(상징적 이름)** 를 통해 값에 안전하게 접근하게 한다.
- 변수는
  - 값을 저장하기 위해 확보한 메모리 공간(또는 그 공간을 가리키는 방법),
  - 그리고 그 공간을 식별하기 위한 이름
  으로 이해할 수 있다.

### 변수 관련 기본 용어

- **변수 이름(변수명)**: 메모리 공간에 붙인 이름 (`result`)
- **변수 값**: 변수에 저장된 값 (`30`)
- **할당(assignment)**: 변수에 값을 저장하는 것
- **참조(reference)**: 변수에 저장된 값을 읽는 것

### 하나의 변수에는 하나의 값

- 변수는 기본적으로 하나의 값만 저장한다.
- 여러 값을 함께 다루려면 배열/객체 같은 자료구조를 사용한다.

```js
var user = { id: 1, name: "Lee" };
var users = [
  { id: 1, name: "Lee" },
  { id: 2, name: "Kim" }
];
```

## 4.2 식별자

- 식별자는 어떤 값을 구별해 식별할 수 있는 고유한 이름이다.
- 변수 이름뿐 아니라 함수 이름, 클래스 이름도 모두 식별자다.
- 식별자는 값을 직접 담는 게 아니라, **값이 저장된 메모리 주소와 연결된 정보**를 바탕으로 값을 찾는다.
- 즉 식별자는 “값의 이름”이라기보다 “메모리 위치를 추적하는 이름”에 가깝다.

## 4.3 변수 선언

- 변수 선언은 변수를 생성하고, 값을 저장할 메모리 공간을 확보하고, 변수 이름과 메모리 공간을 연결하는 과정이다.
- 자바스크립트에서는 `var`, `let`, `const`로 변수를 선언할 수 있다.

```js
var score;
```

- `var` 선언 시 내부적으로
  1) 선언 단계(이름 등록),
  2) 초기화 단계(`undefined` 할당)
  가 함께 진행된다.
- 따라서 `var`로 선언한 변수는 값을 할당하기 전에도 `undefined`를 가진다.
- 선언하지 않은 식별자에 접근하면 `ReferenceError`가 발생한다.

## 4.4 변수 선언의 실행 시점과 호이스팅

- 변수 선언은 코드가 한 줄씩 실행되는 런타임보다 먼저, 소스코드 평가 단계에서 처리된다.
- 그래서 선언문보다 먼저 변수를 참조해도(`var` 기준) 에러 대신 `undefined`가 나올 수 있다.
- 이런 동작을 **호이스팅(hoisting)** 이라고 한다.

```js
console.log(score); // undefined
var score;
```

- `var`, `let`, `const`, `function`, `class` 등 선언문은 모두 호이스팅 대상이다(동작 방식 차이는 있음).

## 4.5 값의 할당

- 할당 연산자 `=`로 값을 저장한다.

```js
var score;    // 선언
score = 80;   // 할당
```

- 아래처럼 한 줄로 써도 내부적으로는 선언과 할당이 분리되어 처리된다.

```js
var score = 80;
```

- 핵심은 **선언은 런타임 이전**, **할당은 런타임 시점**에 실행된다는 점이다.

## 4.6 값의 재할당

- 이미 값이 들어있는 변수에 새 값을 넣는 것을 재할당이라 한다.

```js
var score = 80;
score = 90;
```

- `var`로 선언한 변수는 재할당 가능하다.
- 재할당 시 기존 메모리 공간을 덮는 것이 아니라, 새 메모리 공간에 값을 저장하고 식별자 연결을 바꾸는 방식으로 동작한다.
- 더 이상 참조되지 않는 이전 값은 가비지 컬렉터가 회수한다(회수 시점은 예측 어려움).

### 상수와 `const`

- 재할당이 불가능한 변수가 상수다.
- `const`는 재할당을 금지해 상수 표현에 적합하다.

```js
const foo = 10;
// foo = 100; // TypeError
```

## 4.7 식별자 네이밍 규칙

- 식별자는 문자, 숫자, `_`, `$`를 포함할 수 있다.
- 단, 숫자로 시작할 수 없고 예약어는 사용할 수 없다.
- 자바스크립트는 대소문자를 구분한다(`firstName`과 `firstname`는 다름).
- 의미가 드러나는 이름을 사용해야 가독성과 협업 효율이 올라간다.

### 네이밍 컨벤션

- **camelCase**: 변수, 함수 이름에 주로 사용
- **PascalCase**: 생성자 함수, 클래스 이름에 주로 사용
- **snake_case**, **HungarianCase** 등도 있으나 JS 실무에서는 camelCase/PascalCase가 일반적

---

## 내용 정리

- 변수는 메모리 주소를 직접 다루지 않고 값을 안전하게 저장/참조하게 해주는 추상화 도구다.
- 식별자는 값을 직접 저장하지 않고, 값이 있는 위치를 식별하는 이름이다.
- `var` 변수는 선언 시 `undefined`로 자동 초기화된다.
- 변수 선언은 런타임 이전에 처리되어 호이스팅처럼 동작한다.
- 할당과 재할당은 다르며, 재할당 과정에서 더 이상 참조되지 않는 값은 가비지 컬렉터가 정리한다.
- 좋은 변수명과 일관된 네이밍 컨벤션은 코드 품질의 핵심이다.

## 의문점 제기

1. `var`, `let`, `const`의 호이스팅 차이를 실무에서 어떻게 설명하면 가장 오해가 적을까?
2. 선언과 할당의 실행 시점 차이가 버그로 이어지는 대표 사례는 무엇일까?
3. 가비지 컬렉션 시점을 정확히 모르는 상황에서 성능 이슈를 어떻게 추적해야 할까?
4. “의미 있는 변수명”을 팀 차원에서 강제하려면 어떤 ESLint 규칙/리뷰 기준이 효과적일까?
5. 배열/객체를 변수 하나로 묶는 방식이 과해지는 시점(복잡도 기준)은 어디일까?
6. 초보 학습 단계에서 `var`를 완전히 배제하고 `let/const`만 써도 이해에 문제가 없을까?
7. 상수(`const`) 중심 코딩이 디버깅·리팩터링에 주는 실질적 장점은 어느 정도일까?
8. 네이밍 컨벤션 통일(camelCase, PascalCase)을 자동화하려면 어떤 도구 조합이 가장 간단할까?
