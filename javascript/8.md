## 8. 제어문

제어문은 조건에 따라 코드 블록을 실행하거나 반복 실행할 때 사용한다.

- 조건문: 조건에 따라 코드 블록 실행
- 반복문: 조건이 참인 동안 코드 블록 반복 실행

일반적으로 코드는 위에서 아래로 순차 실행된다. 제어문은 이 흐름을 인위적으로 바꾸므로 강력하지만, 남용하면 코드 흐름을 이해하기 어려워질 수 있다.

## 8.1 블록문

블록문은 0개 이상의 문을 중괄호(`{}`)로 묶은 것이다. 코드 블록 또는 블록이라고도 한다.

```js
{
  var foo = 10;
}

var x = 1;
if (x < 10) {
  x++;
}

function sum(a, b) {
  return a + b;
}
```

- 자바스크립트는 블록문을 하나의 실행 단위로 취급한다.
- 일반적으로 제어문이나 함수 선언에서 사용한다.
- 블록문은 자체 종결성을 가지므로 끝에 세미콜론을 붙이지 않는다.

## 8.2 조건문

조건문은 조건식의 평가 결과에 따라 실행할 코드 블록을 결정한다. 조건식은 불리언 값으로 평가될 수 있는 표현식이다.

자바스크립트의 대표적인 조건문:

- `if...else`
- `switch`

### 8.2.1 if...else 문

`if...else` 문은 조건식의 평가 결과가 참이면 `if` 블록을 실행하고, 거짓이면 `else` 블록을 실행한다.

```js
if (조건식) {
  // 조건식이 true이면 실행
} else {
  // 조건식이 false이면 실행
}
```

조건이 여러 개인 경우 `else if`를 사용할 수 있다.

```js
var num = 2;
var kind;

if (num > 0) {
  kind = "양수";
} else if (num < 0) {
  kind = "음수";
} else {
  kind = "영";
}

console.log(kind); // 양수
```

조건식이 불리언이 아니면 자바스크립트 엔진이 불리언 값으로 암묵적 타입 변환한다.

### if...else와 삼항 조건 연산자

단순히 조건에 따라 값을 선택해 변수에 할당하는 경우에는 삼항 조건 연산자가 더 간결할 수 있다.

```js
var x = 2;
var result = x % 2 ? "홀수" : "짝수";

console.log(result); // 짝수
```

경우의 수가 세 가지여도 중첩 삼항 연산자로 표현할 수는 있다.

```js
var num = 2;
var kind = num ? (num > 0 ? "양수" : "음수") : "영";

console.log(kind); // 양수
```

단, 실행해야 할 코드가 여러 줄이거나 조건이 복잡하면 `if...else`가 더 읽기 좋다.

### 8.2.2 switch 문

`switch` 문은 표현식의 평가 결과와 일치하는 `case` 문으로 실행 흐름을 이동시킨다.

```js
switch (표현식) {
  case 표현식1:
    // 표현식과 표현식1이 일치하면 실행
    break;
  case 표현식2:
    // 표현식과 표현식2가 일치하면 실행
    break;
  default:
    // 일치하는 case가 없으면 실행
}
```

- `if...else`: 참/거짓에 따라 흐름 결정
- `switch`: 여러 값의 경우의 수에 따라 흐름 결정

`switch` 문에서 `break`를 생략하면 다음 `case`까지 이어서 실행된다. 이를 **폴스루(fall through)** 라고 한다.

```js
var month = 11;
var monthName;

switch (month) {
  case 11:
    monthName = "November";
  case 12:
    monthName = "December";
  default:
    monthName = "Invalid month";
}

console.log(monthName); // Invalid month
```

올바르게 종료하려면 각 `case` 끝에 `break`를 사용한다.

```js
switch (month) {
  case 11:
    monthName = "November";
    break;
  case 12:
    monthName = "December";
    break;
  default:
    monthName = "Invalid month";
}
```

폴스루가 의도적으로 유용한 경우도 있다. 여러 `case`를 하나의 조건처럼 묶을 때 사용할 수 있다.

```js
var year = 2000;
var month = 2;
var days = 0;

switch (month) {
  case 1: case 3: case 5: case 7: case 8: case 10: case 12:
    days = 31;
    break;
  case 4: case 6: case 9: case 11:
    days = 30;
    break;
  case 2:
    days = ((year % 4 === 0 && year % 100 !== 0) || (year % 400 === 0)) ? 29 : 28;
    break;
  default:
    console.log("Invalid month");
}

console.log(days); // 29
```

## 8.3 반복문

반복문은 조건식의 평가 결과가 참인 동안 코드 블록을 반복 실행한다.

자바스크립트의 기본 반복문:

- `for`
- `while`
- `do...while`

배열 순회에는 `forEach`, `map`, `filter`, `reduce` 같은 고차 함수도 자주 사용한다.

### 8.3.1 for 문

`for` 문은 조건식이 거짓이 될 때까지 코드 블록을 반복 실행한다.

```js
for (변수 선언문 또는 할당문; 조건식; 증감식) {
  // 조건식이 true이면 반복 실행
}
```

```js
for (var i = 0; i < 2; i++) {
  console.log(i);
}

// 0
// 1
```

실행 흐름:

1. 변수 선언문 또는 할당문 실행(최초 1회)
2. 조건식 평가
3. 조건식이 참이면 코드 블록 실행
4. 증감식 실행
5. 다시 조건식 평가
6. 조건식이 거짓이면 종료

`for` 문의 세 영역은 모두 생략할 수 있지만, 모두 생략하면 무한 루프가 된다.

```js
for (;;) {
  // 무한 루프
}
```

중첩 `for` 문도 사용할 수 있다.

```js
for (var i = 1; i <= 6; i++) {
  for (var j = 1; j <= 6; j++) {
    if (i + j === 6) console.log(`[${i}, ${j}]`);
  }
}
```

### 8.3.2 while 문

`while` 문은 조건식이 참인 동안 코드 블록을 반복 실행한다.

- `for`: 반복 횟수가 비교적 명확할 때 자주 사용
- `while`: 반복 횟수가 불명확할 때 자주 사용

```js
var count = 0;

while (count < 3) {
  console.log(count);
  count++;
}

// 0
// 1
// 2
```

조건식이 항상 참이면 무한 루프가 된다.

```js
while (true) {
  // 무한 루프
}
```

무한 루프에서는 보통 `break`로 탈출 조건을 만든다.

```js
var count = 0;

while (true) {
  console.log(count);
  count++;

  if (count === 3) break;
}
```

### 8.3.3 do...while 문

`do...while` 문은 코드 블록을 먼저 실행한 뒤 조건식을 평가한다. 따라서 코드 블록이 최소 한 번은 실행된다.

```js
var count = 0;

do {
  console.log(count);
  count++;
} while (count < 3);

// 0
// 1
// 2
```

## 8.4 break 문

`break` 문은 반복문, `switch` 문, 레이블 문을 탈출할 때 사용한다.

```js
var string = "Hello World.";
var search = "l";
var index;

for (var i = 0; i < string.length; i++) {
  if (string[i] === search) {
    index = i;
    break;
  }
}

console.log(index); // 2
```

반복문이나 `switch`가 아닌 곳에서 `break`를 사용하면 문법 에러가 발생한다.

```js
if (true) {
  // break; // SyntaxError
}
```

### 레이블 문

레이블 문은 식별자가 붙은 문이다. 중첩 반복문 바깥으로 한 번에 탈출할 때 사용할 수 있다.

```js
outer: for (var i = 0; i < 3; i++) {
  for (var j = 0; j < 3; j++) {
    if (i + j === 3) break outer;
    console.log(`inner [${i}, ${j}]`);
  }
}

console.log("Done!");
```

레이블 문은 흐름을 복잡하게 만들 수 있으므로 중첩 반복문 탈출처럼 필요한 경우에만 제한적으로 사용하는 것이 좋다.

## 8.5 continue 문

`continue` 문은 반복문의 현재 실행을 중단하고 다음 반복으로 넘어간다. `break`처럼 반복문 전체를 탈출하지 않는다.

```js
var string = "Hello World.";
var search = "l";
var count = 0;

for (var i = 0; i < string.length; i++) {
  if (string[i] !== search) continue;
  count++;
}

console.log(count); // 3
```

`continue`는 조건에 맞지 않는 경우를 먼저 걸러내고, 핵심 로직의 들여쓰기를 줄일 때 유용하다.

```js
for (var i = 0; i < string.length; i++) {
  if (string[i] !== search) continue;

  count++;
  // 이후 핵심 로직
}
```

---

## 내용 정리

- 제어문은 조건에 따라 실행 흐름을 바꾸거나 반복 실행을 수행한다.
- 블록문은 여러 문을 하나의 실행 단위로 묶는다.
- `if...else`는 참/거짓 기반 분기에 적합하고, `switch`는 여러 값의 경우의 수를 다룰 때 적합하다.
- `switch`에서 `break`를 생략하면 폴스루가 발생한다.
- `for`는 반복 횟수가 명확할 때, `while`은 반복 횟수가 불명확할 때 자주 사용한다.
- `do...while`은 조건과 무관하게 코드 블록을 최소 한 번 실행한다.
- `break`는 반복문/switch를 탈출하고, `continue`는 현재 반복만 건너뛴다.
- 제어문은 강력하지만 흐름을 복잡하게 만들 수 있으므로 가독성을 고려해 사용해야 한다.

## 의문점 제기

1. `if...else`와 삼항 조건 연산자 중 무엇을 선택할지 정하는 팀 기준은 어떻게 잡으면 좋을까?
2. `switch`의 폴스루는 실무에서 의도적으로 쓰는 것이 좋은가, 아니면 항상 피하는 것이 좋은가?
3. 반복문 대신 `map`, `filter`, `reduce` 같은 고차 함수를 쓰면 항상 가독성이 좋아질까?
4. 중첩 `for` 문은 어느 수준부터 함수 분리나 다른 자료구조로 개선해야 할까?
5. 무한 루프를 사용할 때 안전장치로 어떤 탈출 조건과 로그를 두는 것이 좋을까?
6. `break`와 `continue`를 많이 쓰는 코드는 흐름이 복잡해질 수 있는데, 어떤 리팩터링 기준이 필요할까?
7. 레이블 문은 어떤 상황에서 정당화될 수 있고, 대체로 어떤 방식으로 피할 수 있을까?
8. 조건문이 많아질 때 객체 매핑, 전략 패턴, 함수 분리 중 어떤 방식이 가장 단순한 개선책일까?
