## 10. 객체 리터럴

## 10.1 객체란?

자바스크립트는 객체 기반의 프로그래밍 언어이며, 원시 값을 제외한 거의 모든 값은 객체다.

- 원시 값: 변경 불가능한 값
- 객체: 변경 가능한 값

객체는 0개 이상의 프로퍼티로 구성된 집합이다. 프로퍼티는 **키**와 **값**으로 구성된다.

```js
var person = {
  name: "Lee",
  age: 20
};
```

자바스크립트에서 사용할 수 있는 모든 값은 프로퍼티 값이 될 수 있다. 함수도 값이므로 프로퍼티 값으로 사용할 수 있다. 이때 프로퍼티 값이 함수이면 일반 함수와 구분해 **메서드**라고 부른다.

```js
var counter = {
  num: 0,
  increase: function () {
    this.num++;
  }
};
```

- 프로퍼티: 객체의 상태를 나타내는 값
- 메서드: 프로퍼티를 참조하거나 조작하는 동작

객체는 상태와 동작을 하나의 단위로 구조화할 수 있어 유용하다.

## 10.2 객체 리터럴에 의한 객체 생성

자바스크립트는 프로토타입 기반 객체지향 언어이며 다양한 방식으로 객체를 생성할 수 있다.

- 객체 리터럴
- `Object` 생성자 함수
- 생성자 함수
- `Object.create` 메서드
- 클래스(ES6)

가장 일반적이고 간단한 방식은 **객체 리터럴**이다.

```js
var person = {
  name: "Lee",
  sayHello: function () {
    console.log(`Hello! My name is ${this.name}.`);
  }
};

console.log(typeof person); // object
console.log(person);        // { name: "Lee", sayHello: f }
```

빈 객체도 만들 수 있다.

```js
var empty = {};
console.log(typeof empty); // object
```

객체 리터럴의 `{}`는 코드 블록이 아니라 값으로 평가되는 표현식이다. 따라서 변수에 할당할 때 닫는 중괄호 뒤에 세미콜론을 붙인다.

## 10.3 프로퍼티

객체는 프로퍼티의 집합이며, 프로퍼티는 키와 값으로 구성된다.

```js
var person = {
  name: "Lee",
  age: 20
};
```

프로퍼티 키와 값:

- 프로퍼티 키: 빈 문자열을 포함하는 모든 문자열 또는 심벌 값
- 프로퍼티 값: 자바스크립트에서 사용할 수 있는 모든 값

프로퍼티 키는 보통 문자열이다. 식별자 네이밍 규칙을 만족하면 따옴표를 생략할 수 있다.

```js
var person = {
  firstName: "Ung-mo",
  "last-name": "Lee"
};
```

식별자 네이밍 규칙을 따르지 않는 프로퍼티 키는 반드시 따옴표로 감싸야 한다.

```js
var person = {
  firstName: "Ung-mo",
  // last-name: "Lee" // SyntaxError
  "last-name": "Lee"
};
```

문자열 또는 문자열로 평가할 수 있는 표현식을 사용해 프로퍼티 키를 동적으로 생성할 수도 있다.

```js
var obj = {};
var key = "hello";

obj[key] = "world";

console.log(obj); // { hello: "world" }
```

숫자 리터럴을 프로퍼티 키로 사용하면 내부적으로 문자열로 변환된다.

```js
var foo = {
  0: 1,
  1: 2,
  2: 3
};

console.log(foo); // { 0: 1, 1: 2, 2: 3 }
```

이미 존재하는 프로퍼티 키를 중복 선언하면 나중에 선언한 프로퍼티가 이전 값을 덮어쓴다. 에러는 발생하지 않는다.

```js
var foo = {
  name: "Lee",
  name: "Kim"
};

console.log(foo); // { name: "Kim" }
```

## 10.4 메서드

프로퍼티 값이 함수일 경우 이를 메서드라고 한다. 메서드는 객체에 묶여 있는 함수다.

```js
var circle = {
  radius: 5,
  getDiameter: function () {
    return 2 * this.radius;
  }
};

console.log(circle.getDiameter()); // 10
```

`this`는 메서드를 호출한 객체를 가리키는 참조 변수다. 위 예제에서는 `circle` 객체를 가리킨다.

## 10.5 프로퍼티 접근

프로퍼티에 접근하는 방법은 두 가지다.

- 마침표 표기법: `object.property`
- 대괄호 표기법: `object["property"]`

```js
var person = {
  name: "Lee"
};

console.log(person.name);      // Lee
console.log(person["name"]);   // Lee
```

대괄호 표기법에서 프로퍼티 키는 반드시 문자열로 지정해야 한다. 따옴표를 생략하면 식별자로 해석된다.

```js
var person = {
  name: "Lee"
};

// console.log(person[name]); // ReferenceError
console.log(person["name"]);  // Lee
```

존재하지 않는 프로퍼티에 접근하면 `ReferenceError`가 아니라 `undefined`를 반환한다.

```js
console.log(person.age); // undefined
```

식별자 네이밍 규칙을 따르지 않는 프로퍼티 키는 대괄호 표기법으로 접근해야 한다.

```js
var person = {
  "last-name": "Lee",
  1: 10
};

console.log(person["last-name"]); // Lee
console.log(person[1]);           // 10
console.log(person["1"]);         // 10
```

## 10.6 프로퍼티 값 갱신

이미 존재하는 프로퍼티에 값을 할당하면 프로퍼티 값이 갱신된다.

```js
var person = {
  name: "Lee"
};

person.name = "Kim";

console.log(person); // { name: "Kim" }
```

## 10.7 프로퍼티 동적 생성

존재하지 않는 프로퍼티에 값을 할당하면 프로퍼티가 동적으로 생성되고 값이 할당된다.

```js
var person = {
  name: "Lee"
};

person.age = 20;

console.log(person); // { name: "Lee", age: 20 }
```

## 10.8 프로퍼티 삭제

`delete` 연산자는 객체의 프로퍼티를 삭제한다. 존재하지 않는 프로퍼티를 삭제해도 에러 없이 무시된다.

```js
var person = {
  name: "Lee"
};

person.age = 20;

delete person.age;
delete person.address;

console.log(person); // { name: "Lee" }
```

## 10.9 ES6에서 추가된 객체 리터럴 확장 기능

ES6에서는 객체 리터럴을 더 간결하게 작성할 수 있는 기능이 추가되었다.

### 10.9.1 프로퍼티 축약 표현

프로퍼티 값으로 변수를 사용할 때, 변수 이름과 프로퍼티 키가 같으면 프로퍼티 키를 생략할 수 있다.

```js
// ES5
var x = 1;
var y = 2;

var obj = {
  x: x,
  y: y
};

console.log(obj); // { x: 1, y: 2 }
```

```js
// ES6
let x = 1;
let y = 2;

const obj = { x, y };

console.log(obj); // { x: 1, y: 2 }
```

### 10.9.2 계산된 프로퍼티 이름

표현식을 사용해 프로퍼티 키를 동적으로 만들 수 있다. 이때 표현식을 대괄호로 감싼다.

```js
// ES5
var prefix = "prop";
var i = 0;
var obj = {};

obj[prefix + "-" + ++i] = i;
obj[prefix + "-" + ++i] = i;
obj[prefix + "-" + ++i] = i;

console.log(obj); // { prop-1: 1, prop-2: 2, prop-3: 3 }
```

ES6에서는 객체 리터럴 내부에서도 계산된 프로퍼티 이름을 사용할 수 있다.

```js
const prefix = "prop";
let i = 0;

const obj = {
  [`${prefix}-${++i}`]: i,
  [`${prefix}-${++i}`]: i,
  [`${prefix}-${++i}`]: i
};

console.log(obj); // { prop-1: 1, prop-2: 2, prop-3: 3 }
```

### 10.9.3 메서드 축약 표현

ES5에서는 메서드를 프로퍼티 값으로 함수 표현식을 할당해 정의했다.

```js
var obj = {
  name: "Lee",
  sayHi: function () {
    console.log("Hi! " + this.name);
  }
};

obj.sayHi(); // Hi! Lee
```

ES6에서는 `function` 키워드를 생략한 메서드 축약 표현을 사용할 수 있다.

```js
const obj = {
  name: "Lee",
  sayHi() {
    console.log("Hi! " + this.name);
  }
};

obj.sayHi(); // Hi! Lee
```

ES6 메서드 축약 표현으로 정의한 메서드는 일반 프로퍼티에 할당한 함수와 내부 동작이 일부 다르므로 이후 장에서 더 자세히 다룬다.

---

## 내용 정리

- 객체는 프로퍼티의 집합이며, 프로퍼티는 키와 값으로 구성된다.
- 프로퍼티 값이 함수이면 메서드라고 부른다.
- 객체 리터럴은 가장 간단한 객체 생성 방식이며 값으로 평가되는 표현식이다.
- 프로퍼티 키는 문자열 또는 심벌 값이며, 식별자 네이밍 규칙을 만족하면 따옴표를 생략할 수 있다.
- 프로퍼티 접근은 마침표 표기법과 대괄호 표기법을 사용한다.
- 존재하지 않는 프로퍼티에 접근하면 `undefined`가 반환된다.
- 존재하지 않는 프로퍼티에 값을 할당하면 동적으로 생성된다.
- ES6에서는 프로퍼티 축약, 계산된 프로퍼티 이름, 메서드 축약 표현을 사용할 수 있다.

## 의문점 제기

1. 객체의 프로퍼티 키를 문자열로 관리할 때, 마침표 표기법과 대괄호 표기법의 선택 기준은 무엇일까?
2. 존재하지 않는 프로퍼티 접근이 에러가 아니라 `undefined`를 반환하는 특성은 어떤 버그를 만들 수 있을까?
3. 프로퍼티를 동적으로 추가하는 방식은 언제 유용하고, 언제 객체 구조를 불명확하게 만들까?
4. `delete` 연산자 대신 값을 `null`로 두거나 새 객체를 만드는 방식이 더 나은 경우는 언제일까?
5. 메서드 내부의 `this`는 호출 방식에 따라 달라지는데, 이를 안전하게 이해하려면 어떤 예제를 더 봐야 할까?
6. 프로퍼티 키 중복 선언이 에러를 내지 않는 점을 방지하려면 어떤 린트 규칙이 필요할까?
7. ES6 메서드 축약 표현과 일반 함수 프로퍼티는 내부 동작이 어떻게 다를까?
8. 객체 리터럴이 커질 때 함수 분리, 클래스, 팩토리 함수 중 어떤 방식으로 구조화하는 것이 좋을까?

## 답변

1. **마침표 vs 대괄호 선택 기준**
   - **마침표 표기법**: 키가 **고정된 식별자 형태**(예: `obj.name`)이고, 코드에서 바로 읽히는 게 중요한 경우.
   - **대괄호 표기법**: 키가 **동적**이거나(예: `obj[key]`), **식별자 규칙을 깨는 키**(예: `obj["last-name"]`)거나, 숫자/심벌 등 **문자열 변환 규칙을 의식**해야 하는 경우.
   - 결론적으로 “키가 컴파일 타임에 정해져 있으면 점, 런타임에 정해지거나 특수 키면 대괄호”로 잡으면 안전하다.

2. **없는 프로퍼티가 `undefined`를 반환해서 생기는 버그**
   - **오타가 조용히 통과**한다: `user.nmae` 같은 실수가 즉시 터지지 않고 `undefined`로 흘러가 디버깅이 늦어진다.
   - **분기 로직 오작동**: `if (obj.prop)` 같은 진리값 체크에서 의도치 않게 false로 처리된다(값이 `0`, `""`, `false`인 경우도 함께 걸림).
   - **연쇄 접근에서 런타임 에러**: `obj.a.b`에서 `obj.a`가 `undefined`면 그 다음에 `TypeError`가 난다.
   - 대응은 보통 **옵셔널 체이닝**(`obj.a?.b`), **명시적 존재 검사**(`"prop" in obj`, `Object.hasOwn(obj,"prop")`)를 쓴다.

3. **동적 추가가 유용한 때 vs 구조를 흐리는 때**
   - **유용**: 키가 데이터에서 오거나(서버 응답을 키로 매핑), 옵션 플래그를 누적하거나, 설정 객체를 조합할 때(예: 조건부로 프로퍼티를 추가).
   - **문제**: 객체가 “어떤 형태를 갖는지”가 코드만 보고 파악이 어려워지고, 오타/누락이 **정적 분석으로 잡히기 어렵다**.
   - 실무 팁: 동적 키를 쓰더라도 “허용 키 목록”을 두거나, 생성 시점에 한 번에 만들고(immutable 스타일), 이후엔 형태를 고정하는 편이 유지보수에 유리하다.

4. **`delete` 대신 `null` 또는 새 객체가 더 나은 경우**
   - **`null`**: “없음”을 **명시적으로 표현**하고 싶을 때(프로퍼티는 존재하지만 값이 비어있음을 의미). API 스키마/직렬화에서 구분이 필요할 때도 있다.
   - **새 객체(복사 후 제외)**: 상태 관리(React/Redux 등)처럼 **불변성을 선호**하거나, 예측 가능한 변경(추적/디버깅)을 원할 때.
   - **`delete`**: 정말로 키 자체를 제거해야 하거나(사전/맵처럼 쓰는 객체), 메모리/직렬화 결과에서 키를 없애야 할 때 의미가 있다.

5. **`this`를 안전하게 이해하기 위한 예제(핵심 케이스)**
   - **메서드 호출**: `obj.method()`에서 `this === obj`.
   - **함수로 분리 호출**: `const f = obj.method; f()`에서 `this`가 달라진다(엄격 모드에서는 `undefined`).
   - **콜백 전달**: `setTimeout(obj.method, 0)`처럼 전달하면 위와 같은 문제가 재현된다.
   - **해결 패턴**: `bind`, 래핑 함수 `() => obj.method()`, 또는 “상태를 캡슐화한 클로저/클래스” 중 상황에 맞는 걸 선택한다.

6. **프로퍼티 키 중복 선언 방지 린트**
   - ESLint를 쓴다면 객체 리터럴의 중복 키를 막는 규칙으로 **`no-dupe-keys`**를 켜는 게 일반적이다.
   - (추가로) 타입스크립트라면 객체 타입을 명확히 두면 키 오타/중복을 더 일찍 잡는 데 도움이 된다.

7. **ES6 메서드 축약 vs 함수 프로퍼티의 내부 동작 차이(핵심만)**
   - 둘 다 “호출하면 함수가 실행된다”는 점은 같지만, **메서드 축약으로 정의한 함수는 `[[HomeObject]]`를 가져서 `super`를 사용할 수 있다**(객체 리터럴/클래스 문맥에서).
   - 반면 `sayHi: function(){}` 형태의 일반 함수 프로퍼티는 그 문맥의 메서드 정의로 취급되지 않아 `super`와의 연결이 다르다.
   - (참고) 둘 다 화살표 함수와는 다르게 `this`가 “호출 방식”에 의해 결정된다.

8. **객체 리터럴이 커질 때 구조화 선택(권장 기준)**
   - **함수 분리(모듈 함수)**: 상태가 크지 않고, 순수 함수/유틸 중심이면 가장 단순하고 테스트가 쉽다.
   - **팩토리 함수**: 인스턴스를 여러 개 만들고, “클래스까지는 과한데 상태+행동을 묶고 싶을 때” 유용하다(클로저로 캡슐화도 가능).
   - **클래스**: 동일한 형태의 인스턴스를 많이 만들고, 상속/인터페이스/명확한 생명주기(생성/초기화)가 필요하면 적합하다.
   - 기준은 “인스턴스 수(1개 vs 여러 개)”, “상태 캡슐화 필요성”, “테스트/교체 용이성”으로 잡으면 선택이 빨라진다.
