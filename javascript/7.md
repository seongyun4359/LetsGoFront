## 7. 연산자

연산자는 하나 이상의 표현식을 대상으로 산술, 할당, 비교, 논리, 타입, 지수 연산 등을 수행해 하나의 값을 만든다.  
연산의 대상이 되는 값을 **피연산자**라고 한다.

```js
5 * 4;                    // 산술 연산자
"My name is " + "Lee";    // 문자열 연결 연산자
color = "red";            // 할당 연산자
3 > 5;                    // 비교 연산자
true && false;            // 논리 연산자
typeof "Hi";              // 타입 연산자
```

- 피연산자: 값으로 평가될 수 있는 표현식
- 연산자 표현식: 피연산자와 연산자의 조합으로 만들어진 표현식

## 7.1 산술 연산자

산술 연산자는 피연산자를 계산해 새로운 숫자 값을 만든다. 계산할 수 없으면 `NaN`을 반환한다.

### 이항 산술 연산자

- 피연산자 2개를 계산한다.
- 피연산자의 값을 직접 변경하지 않는다.
- 즉, 부수 효과가 없다.

| 연산자 | 의미 |
|--------|------|
| `+` | 덧셈 |
| `-` | 뺄셈 |
| `*` | 곱셈 |
| `/` | 나눗셈 |
| `%` | 나머지 |

```js
5 + 2; // 7
5 - 2; // 3
5 * 2; // 10
5 / 2; // 2.5
5 % 2; // 1
```

### 단항 산술 연산자

- 피연산자 1개를 대상으로 연산한다.

| 연산자 | 의미 | 부수 효과 |
|--------|------|-----------|
| `++` | 증가 | 있음 |
| `--` | 감소 | 있음 |
| `+` | 숫자 타입 변환 | 없음 |
| `-` | 부호 반전/숫자 타입 변환 | 없음 |

```js
var x = 1;

x++;
console.log(x); // 2

x--;
console.log(x); // 1
```

`++`, `--`는 위치에 따라 동작 시점이 달라진다.

```js
var x = 5;
var result;

result = x++; // 먼저 할당, 나중에 증가
console.log(result, x); // 5 6

result = ++x; // 먼저 증가, 나중에 할당
console.log(result, x); // 7 7
```

단항 `+`는 숫자가 아닌 값을 숫자로 변환한 결과를 반환한다.

```js
+"1";       // 1
+true;      // 1
+false;     // 0
+"Hello";   // NaN
```

단항 `-`는 숫자로 변환한 뒤 부호를 반전한다.

```js
-"10";    // -10
-true;    // -1
-"Hello"; // NaN
```

### 문자열 연결 연산자

`+` 연산자는 피연산자 중 하나 이상이 문자열이면 문자열 연결 연산자로 동작한다.

```js
"1" + 2;      // "12"
1 + "2";      // "12"
1 + 2;        // 3
1 + true;     // 2
1 + false;    // 1
1 + null;     // 1
1 + undefined; // NaN
```

이처럼 자바스크립트는 연산 과정에서 암묵적 타입 변환을 수행할 수 있다.

## 7.2 할당 연산자

할당 연산자는 우항의 평가 결과를 좌항의 변수에 할당한다. 변수 값이 바뀌므로 부수 효과가 있다.

| 연산자 | 동일 표현 |
|--------|-----------|
| `=` | `x = 5` |
| `+=` | `x = x + 5` |
| `-=` | `x = x - 5` |
| `*=` | `x = x * 5` |
| `/=` | `x = x / 5` |
| `%=` | `x = x % 5` |

```js
var x;
x = 10;
x += 5; // 15
x -= 5; // 10
x *= 5; // 50
x /= 5; // 10
x %= 5; // 0
```

할당문은 표현식인 문이다. 즉, 할당된 값으로 평가된다.

```js
var x;
console.log(x = 10); // 10

var a, b, c;
a = b = c = 0;
console.log(a, b, c); // 0 0 0
```

## 7.3 비교 연산자

비교 연산자는 좌항과 우항을 비교해 불리언 값을 반환한다. 주로 조건식에서 사용한다.

### 동등/일치 비교 연산자

| 연산자 | 의미 | 특징 |
|--------|------|------|
| `==` | 동등 비교 | 암묵적 타입 변환 후 비교 |
| `===` | 일치 비교 | 타입과 값 모두 비교 |
| `!=` | 부동등 비교 | `==`의 반대 |
| `!==` | 불일치 비교 | `===`의 반대 |

```js
5 == "5";   // true
5 === "5";  // false
```

`==`는 결과 예측이 어려워 실무에서는 보통 `===` 사용을 권장한다.

```js
false == "0"; // true
0 == "";      // true
```

`NaN`은 자기 자신과도 일치하지 않는 유일한 값이다.

```js
NaN === NaN; // false
Number.isNaN(NaN); // true
```

`+0`과 `-0`, `NaN`까지 정확히 비교하려면 `Object.is`를 사용할 수 있다.

```js
0 === -0; // true
Object.is(0, -0); // false

NaN === NaN; // false
Object.is(NaN, NaN); // true
```

### 대소 관계 비교 연산자

| 연산자 | 의미 |
|--------|------|
| `>` | 크다 |
| `<` | 작다 |
| `>=` | 크거나 같다 |
| `<=` | 작거나 같다 |

```js
5 > 0;  // true
5 > 5;  // false
5 >= 5; // true
5 <= 5; // true
```

## 7.4 삼항 조건 연산자

삼항 조건 연산자는 조건식의 평가 결과에 따라 반환할 값을 결정한다.

```js
조건식 ? 조건식이 true일 때 반환할 값 : 조건식이 false일 때 반환할 값
```

```js
var score = 80;
var result = score >= 60 ? "pass" : "fail";
console.log(result); // pass
```

- 삼항 조건 연산자는 표현식이다.
- 따라서 값처럼 변수에 할당할 수 있다.
- 단순히 값을 고르는 경우에는 유용하지만, 실행할 문이 많다면 `if...else`가 더 읽기 좋다.

## 7.5 논리 연산자

논리 연산자는 논리 연산 결과를 반환한다.

| 연산자 | 의미 |
|--------|------|
| `||` | OR |
| `&&` | AND |
| `!` | NOT |

```js
true || false;  // true
true && false;  // false
!true;          // false
```

`!` 연산자는 항상 불리언 값을 반환한다. 피연산자가 불리언이 아니면 불리언으로 암묵적 타입 변환한다.

```js
!0;       // true
!"Hello"; // false
```

`||`, `&&`의 결과는 반드시 불리언이 아닐 수 있다. 두 피연산자 중 하나로 평가된다.

```js
"Cat" && "Dog"; // "Dog"
```

## 7.6 쉼표 연산자

쉼표 연산자는 왼쪽부터 차례대로 평가하고, 마지막 피연산자의 평가 결과를 반환한다.

```js
var x, y, z;
x = 1, y = 2, z = 3; // 3
```

## 7.7 그룹 연산자

그룹 연산자 `()`는 감싼 표현식을 가장 먼저 평가한다. 연산자 우선순위를 명시적으로 조절할 때 사용한다.

```js
10 * 2 + 3;   // 23
10 * (2 + 3); // 50
```

## 7.8 typeof 연산자

`typeof` 연산자는 피연산자의 데이터 타입을 문자열로 반환한다.

```js
typeof "";          // "string"
typeof 1;           // "number"
typeof NaN;         // "number"
typeof true;        // "boolean"
typeof undefined;   // "undefined"
typeof Symbol();    // "symbol"
typeof null;        // "object"
typeof [];          // "object"
typeof {};          // "object"
typeof function(){}; // "function"
```

주의할 점:

- `typeof null`은 `"object"`를 반환한다. 오래된 버그지만 호환성 때문에 유지된다.
- `null` 확인에는 `typeof`가 아니라 `=== null`을 사용한다.
- 선언하지 않은 식별자에 `typeof`를 사용하면 `ReferenceError`가 아니라 `"undefined"`가 반환된다.

```js
var foo = null;

typeof foo === "object"; // true
foo === null;            // true

typeof undeclared;       // "undefined"
```

## 7.9 지수 연산자

지수 연산자 `**`는 좌항을 밑, 우항을 지수로 거듭제곱한다.

```js
2 ** 2;    // 4
2 ** 0;    // 1
2 ** -2;   // 0.25
```

기존에는 `Math.pow`를 사용했다.

```js
Math.pow(2, 2); // 4
```

지수 연산자는 우결합성을 가진다.

```js
2 ** (3 ** 2); // 512
```

음수를 밑으로 사용할 때는 괄호가 필요하다.

```js
(-5) ** 2; // 25
```

## 7.10 그 외의 연산자

아래 연산자들은 관련 주제에서 자세히 다룬다.

| 연산자 | 개요 |
|--------|------|
| `?.` | 옵셔널 체이닝 |
| `??` | null 병합 |
| `delete` | 프로퍼티 삭제 |
| `new` | 생성자 함수 호출 |
| `instanceof` | 인스턴스 여부 판별 |
| `in` | 프로퍼티 존재 확인 |

## 7.11 연산자의 부수 효과

대부분의 연산자는 새로운 값을 만들 뿐 다른 코드에 영향을 주지 않는다.  
하지만 일부 연산자는 피연산자나 객체 상태를 변경한다.

부수 효과가 있는 연산자:

- 할당 연산자 `=`
- 증가/감소 연산자 `++`, `--`
- `delete` 연산자

```js
var x = 1;
x++;
console.log(x); // 2

var obj = { a: 1 };
delete obj.a;
console.log(obj); // {}
```

## 7.12 연산자 우선순위

연산자 우선순위는 여러 연산자가 있을 때 어떤 연산자가 먼저 실행되는지를 정한다.

- `()` 그룹 연산자의 우선순위가 가장 높다.
- `**`는 이항 연산자 중 우선순위가 가장 높다.
- 우선순위를 모두 외우기보다, 헷갈리는 경우 `()`로 명시하는 것이 좋다.

```js
10 * (2 + 3); // 50
```

## 7.13 연산자 결합 순서

연산자 결합 순서는 같은 우선순위의 연산자를 어느 방향부터 평가할지 정한다.

- 대부분의 이항 연산자: 좌항에서 우항
- 할당 연산자, 단항 연산자, 삼항 조건 연산자, 지수 연산자: 우항에서 좌항

```js
var a, b, c;
a = b = c = 1; // 오른쪽부터 평가
```

---

## 내용 정리

- 연산자는 피연산자를 대상으로 연산해 새로운 값을 만든다.
- 산술 연산자는 숫자 값을 만들며, 계산 불가능하면 `NaN`을 반환한다.
- `++`, `--`, 할당 연산자, `delete`는 부수 효과가 있다.
- `+`는 숫자 덧셈 또는 문자열 연결로 동작하며, 암묵적 타입 변환을 유발할 수 있다.
- 비교에는 보통 `==`보다 `===`를 사용하는 것이 안전하다.
- `typeof null`은 `"object"`이므로 `null` 판별에는 `=== null`을 사용한다.
- 삼항 조건 연산자는 값으로 평가되는 표현식이고, `if...else`는 문이다.
- 연산자 우선순위가 헷갈리면 `()`로 의도를 명확히 표현하는 것이 좋다.

## 의문점 제기

1. `==`를 완전히 금지하는 것이 항상 좋은가, 아니면 의도적 타입 변환을 활용할 수 있는 상황이 있을까?
2. `++`, `--`처럼 부수 효과가 있는 연산자는 코드 리뷰에서 어떤 기준으로 허용해야 할까?
3. `+` 연산자의 문자열 연결과 숫자 덧셈이 섞일 때, 버그를 줄이기 위한 팀 규칙은 무엇이 좋을까?
4. `Number.isNaN`, `Object.is`, `===`는 각각 어떤 비교 상황에서 선택해야 할까?
5. 삼항 조건 연산자를 중첩해서 쓰는 것은 어느 수준부터 가독성을 해칠까?
6. `&&`, `||`가 불리언이 아닌 값을 반환하는 단축 평가는 실무에서 어떤 패턴으로 자주 쓰일까?
7. 연산자 우선순위는 외우는 것보다 괄호를 쓰는 것이 낫다면, 괄호를 어디까지 명시하는 것이 적절할까?
8. `delete` 연산자는 객체 shape/성능에 영향을 줄 수 있는데, 실무에서는 어떤 대안을 고려할 수 있을까?
