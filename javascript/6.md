## 6. 데이터 타입

데이터 타입은 값의 종류를 말한다. 자바스크립트의 모든 값은 데이터 타입을 가진다.

자바스크립트의 타입은 크게 **원시 타입**과 **객체 타입**으로 나뉜다.

| 구분 | 데이터 타입 | 설명 |
|------|-------------|------|
| 원시 타입 | number | 정수와 실수를 구분하지 않는 숫자 타입 |
| 원시 타입 | string | 문자열 |
| 원시 타입 | boolean | `true`, `false` |
| 원시 타입 | undefined | 값이 할당되지 않은 변수의 기본값 |
| 원시 타입 | null | 값이 없음을 의도적으로 명시 |
| 원시 타입 | symbol | ES6에서 추가된 유일무이한 값 |
| 원시 타입 | bigint | ES2020에서 추가된 큰 정수 타입 |
| 객체 타입 | object | 객체, 배열, 함수 등 |

## 6.1 숫자 타입

- 자바스크립트는 정수와 실수를 구분하지 않고 하나의 `number` 타입만 사용한다.
- 모든 숫자는 배정밀도 64비트 부동소수점 형식으로 처리된다.
- 따라서 정수처럼 보여도 내부적으로는 실수로 처리된다.

```js
var integer = 10;
var double = 10.12;
var negative = -20;

console.log(1 === 1.0); // true
console.log(3 / 2);     // 1.5
```

숫자 타입의 특별한 값:

- `Infinity`: 양의 무한대
- `-Infinity`: 음의 무한대
- `NaN`: 산술 연산 불가(Not-a-Number)

```js
console.log(10 / 0);        // Infinity
console.log(10 / -0);       // -Infinity
console.log(1 * "string");  // NaN
```

## 6.2 문자열 타입

- 문자열 타입은 텍스트 데이터를 표현한다.
- 작은따옴표, 큰따옴표, 백틱으로 감싼다.
- 따옴표로 감싸지 않으면 식별자로 해석된다.
- 자바스크립트의 문자열은 원시 타입이며 변경 불가능한 값이다.

```js
var str1 = "문자열";
var str2 = '문자열';
var str3 = `문자열`;
```

## 6.3 템플릿 리터럴

- ES6에서 도입된 문자열 표기법이다.
- 백틱(`` ` ``)을 사용한다.
- 멀티라인 문자열, 표현식 삽입 등을 쉽게 처리할 수 있다.

```js
var first = "Ung-mo";
var last = "Lee";

console.log(`My name is ${first} ${last}.`);
```

### 멀티라인 문자열

일반 문자열은 줄바꿈을 직접 포함할 수 없지만, 템플릿 리터럴은 줄바꿈과 공백을 그대로 보존한다.

```js
var html = `<ul>
  <li><a href="#">Home</a></li>
</ul>`;
```

### 표현식 삽입

`${}` 안에 표현식을 넣으면 평가 결과가 문자열로 변환되어 삽입된다.

```js
console.log(`1 + 2 = ${1 + 2}`); // 1 + 2 = 3
```

## 6.4 불리언 타입

- 불리언 타입은 논리적 참/거짓을 나타낸다.
- 값은 `true`, `false` 두 개뿐이다.
- 조건문과 흐름 제어에서 자주 사용된다.

```js
var done = true;
done = false;
```

## 6.5 undefined 타입

- `undefined` 타입의 값은 `undefined` 하나뿐이다.
- `var`로 선언한 변수는 값이 할당되기 전 `undefined`로 초기화된다.
- 개발자가 의도적으로 `undefined`를 할당하는 것은 권장하지 않는다.
- “값이 없음”을 명시하려면 보통 `null`을 사용한다.

```js
var foo;
console.log(foo); // undefined
```

## 6.6 null 타입

- `null` 타입의 값은 `null` 하나뿐이다.
- `null`은 값이 없다는 것을 **의도적으로 명시**할 때 사용한다.
- 함수가 유효한 값을 반환할 수 없을 때 `null`을 반환하기도 한다.

```js
var foo = "Lee";
foo = null;

var element = document.querySelector(".myClass");
console.log(element); // 요소가 없으면 null
```

## 6.7 심벌 타입

- `symbol`은 ES6에서 추가된 원시 타입이다.
- 다른 값과 중복되지 않는 유일무이한 값을 만든다.
- 주로 객체의 고유한 프로퍼티 키를 만들 때 사용한다.

```js
var key = Symbol("key");
var obj = {};

obj[key] = "value";
console.log(obj[key]); // value
```

## 6.8 객체 타입

- 자바스크립트는 객체 기반 언어다.
- 원시 타입을 제외한 값은 모두 객체 타입이다.
- 객체, 배열, 함수 등이 객체 타입에 해당한다.

## 6.9 데이터 타입의 필요성

데이터 타입이 필요한 이유:

1. 값을 저장할 때 확보해야 하는 메모리 공간의 크기를 결정하기 위해
2. 값을 참조할 때 한 번에 읽어야 할 메모리 크기를 결정하기 위해
3. 메모리에서 읽은 2진수를 어떻게 해석할지 결정하기 위해

예를 들어 같은 `0100 0001`이라는 비트도 숫자로 해석하면 `65`, 문자로 해석하면 `"A"`가 될 수 있다.  
따라서 타입은 메모리에 저장된 값을 어떤 의미로 해석할지 정하는 기준이다.

## 6.10 동적 타이핑

자바스크립트는 동적 타입 언어다.

- 변수 선언 시 타입을 지정하지 않는다.
- 값이 할당되는 시점에 타입이 결정된다.
- 재할당에 의해 변수의 타입이 언제든 바뀔 수 있다.

```js
var foo;

foo = 3;
console.log(typeof foo); // number

foo = "Hello";
console.log(typeof foo); // string

foo = true;
console.log(typeof foo); // boolean
```

정확히는 “변수의 타입”이 바뀐다기보다, 변수에 할당된 **값의 타입**이 바뀐다고 이해하는 것이 좋다.

### 동적 타입 언어의 주의점

- 편리하지만 값의 타입을 예측하기 어려워질 수 있다.
- 암묵적 타입 변환 때문에 의도하지 않은 결과가 생길 수 있다.
- 변수는 필요한 만큼만 사용한다.
- 변수의 스코프는 좁게 유지한다.
- 전역 변수는 최대한 피한다.
- 재할당보다는 `const`를 우선 사용한다.
- 변수명은 의미가 드러나게 작성한다.

---

## 내용 정리

- 자바스크립트의 값은 모두 데이터 타입을 가진다.
- 타입은 원시 타입과 객체 타입으로 나뉜다.
- 숫자는 하나의 `number` 타입으로 처리되며, `Infinity`, `-Infinity`, `NaN` 같은 특별한 값도 있다.
- 문자열은 원시 타입이며 변경 불가능하다.
- 템플릿 리터럴은 멀티라인 문자열과 표현식 삽입을 쉽게 만든다.
- `undefined`는 엔진이 초기화에 사용하는 값이고, `null`은 값이 없음을 의도적으로 표현하는 값이다.
- `symbol`은 충돌 없는 고유한 값을 만들 때 사용한다.
- 자바스크립트는 동적 타입 언어라 값에 따라 변수의 타입이 런타임에 결정된다.

## 의문점 제기

1. `undefined`와 `null`을 실무에서 어떻게 구분해 사용하는 것이 가장 명확할까?
2. `typeof null`이 `"object"`를 반환하는 특성은 왜 남아 있으며, 이를 어떻게 방어적으로 다뤄야 할까?
3. 동적 타입 언어의 유연성과 타입 안정성 사이에서 TypeScript는 어떤 문제를 해결해 주는가?
4. `NaN`은 자기 자신과도 같지 않은데, 이를 안전하게 검사하려면 어떤 방법을 써야 할까?
5. 템플릿 리터럴을 남용하면 오히려 가독성이 떨어지는 경우는 언제일까?
6. `symbol`은 실무 코드에서 언제 직접 사용하는 것이 적절할까?
7. `number`의 부동소수점 오차는 금융/정산 코드에서 어떻게 처리해야 할까?
8. 변수의 스코프를 좁게 유지하는 구체적인 리팩터링 방법에는 무엇이 있을까?
